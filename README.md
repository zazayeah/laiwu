# 莱芜重卡充电运营分析系统aaaaa

面向重卡充电场景的精细化运营分析与辅助决策系统，围绕数据治理、负荷认知、用户运营、柔性调节、辅助决策和空间选址建设，形成从分析到决策输出的完整业务闭环。

本仓库当前以原型演示和联调验证为主，已经具备前端演示、Java 接口网关和 Python 算法服务三部分代码结构，适合继续完善为可联调、可部署、可协作的项目。

## 项目目标

- 汇聚站点台账、充电记录、区域负荷、地图空间和用户行为等多源数据
- 构建重卡充电负荷特征画像和区域/站点负荷预测能力
- 识别典型重卡用户群体，分析站点偏好、补能窗口和未来行为
- 评估柔性调节潜力，支撑移峰填谷和夜间补能优化
- 形成街道预警、站点策略、AI 充电次序优化和空间选址建议
- 支撑充电用户、充电站和供电公司三类对象的分层应用

## 业务链条

系统当前围绕以下 6 条业务链条展开：

1. 数据汇聚治理链
2. 负荷认知与预测链
3. 用户运营分析链
4. 柔性调节执行链
5. 预警处置闭环链
6. 空间选址建设链

对应的系统主线可以概括为：

`识别和预测 -> 分析与优化 -> 决策与输出`

## 子项目说明

### 1. `laiwu/`

静态 HTML 原型文件，适合快速展示页面逻辑和业务流程。

适用场景：

- 汇报演示
- 快速查看原型结构
- 与 Vue 版本做页面对照

### 2. `laiwu_vue版/`

主前端项目，基于 Vue 3 + Vite + Pinia + Vue Router + ECharts 搭建。

主要页面包括：

- 总览驾驶舱
- 多源数据整合与治理
- 多维度负荷特征画像构建
- 重卡用户群体画像与行为预测
- 调节潜力分析与量化评估
- 充电负荷预测
- 辅助决策数据应用
- 地图充电站管理 / 地图选址分析

前端默认可直接使用本地模拟数据运行，无需后端即可完成原型演示。

### 3. `laiwu_backend_java/`

Java 后端骨架，承担场景数据接口和算法网关能力。

主要作用：

- 提供前端场景接口
- 对接 Python 算法服务
- 承载后续统一接口出口

### 4. `laiwu_algo_python/`

Python 算法服务骨架，承担负荷画像、用户画像、调节评估和负荷预测等算法接口。

主要作用：

- 提供算法计算接口
- 为 Java 网关提供下游服务
- 支撑后续真实模型接入

## 快速开始

### 方式一：仅前端演示

如果你的目标是演示原型或答辩展示，只需要启动前端即可。

```bash
cd laiwu_vue版
npm install
npm run dev
```

说明：

- 不配置 `VITE_API_BASE_URL` 时，前端会自动回退本地模拟数据
- 这种模式下不依赖 Java 后端和 Python 算法服务

### 方式二：前后端联调

#### 启动 Python 算法服务

```bash
cd laiwu_algo_python
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

#### 启动 Java 后端

```bash
cd laiwu_backend_java
mvn spring-boot:run
```

#### 启动前端并接入后端

```bash
cd laiwu_vue版
VITE_API_BASE_URL=http://localhost:8080/api npm run dev
```
